WasmExtension

WasmExtensions provides a mechanism to extend the functionality provided by the Istio proxy through Web Assembly filters.

Field Type Description Required
config Struct

The configuration that will be passed on to the extension.

 No
image string

URL of a compatible OCI image.

 No
imagePullPolicy string

The pull behaviour to be applied when fetching the image. Can be one of IfNotPresent, Always. Only relevant when images are referenced by tag. If set to Always, istiod will always pull the latest version of an image when applying this WasmExtension. If set to IfNotPresent, istiod will use an existing version of an image if it has been pulled before. If not, it will pull the latest version.

 No
imagePullSecrets string[]

Credentials to use for image pulling. List of K8s Secrets that contain docker pull secrets which are to be used to authenticate against the registry when pulling the image. If the name contains a forward-slash, it will be interpreted as /. Names that do not contain forward-slashes are interpreted as the names of secrets in the same namespace as the WasmExtension.

 No
workloadSelector WorkloadSelector

Criteria used to select the specific set of pods/VMs on which this Extension configuration should be applied. If omitted, this configuration will be applied to all workload instances in the same namespace. If omitted, the WasmExtension will be applied to all workloads in the same namespace. If the WasmExtension is present in the config root namespace, it will be applied to all applicable workloads in any namespace.

 No
phase FilterClass

Determines where in the filter chain this WasmExtension is to be injected.

 No
priority Int64Value

Determines ordering of WasmExtensions in the same phase. When multiple WasmExtensions are applied to the same workload in the same phase, they will be applied by priority, in descending order. If priority is not set, or two WasmExtensions exist with the same value, the ordering will be deterministically derived from name and namespace of the WasmExtensions.

 No
