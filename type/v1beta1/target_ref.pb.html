PolicyTargetReference

PolicyTargetReferences specifies the targeted resource which the policy can be applied to. It must only target a single resource at a time, but it can be used to target larger resources such as Gateways that may apply to multiple child resources. The PolicyTargetReference will replace the WorkloadSelector for AuthorizationPolicies applied to ambient workloads.

Field Type Description Required
group string

group is the group of the target resource.

+kubebuilder:validation:MinLength=1 +kubebuilder:validation:MaxLength=253

 No
kind string

kind is kind of the target resource.

+kubebuilder:validation:MinLength=1 +kubebuilder:validation:MaxLength=253

 No
name string

name is the name of the target resource.

+kubebuilder:validation:MinLength=1 +kubebuilder:validation:MaxLength=253

 No
namespace string (oneof)

namespace is the namespace of the referent. When unspecified, the local namespace is inferred. Even when policy targets a resource in a different namespace, it may only apply to traffic originating from the same namespace as the policy.

+kubebuilder:validation:MinLength=1 +kubebuilder:validation:MaxLength=253 +optional
